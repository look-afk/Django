Vacancy API Dashboard

Feature your Django job hunting platform with a clean frontend preview and built-in API usage guide. Use these routes to create, read, update, and delete vacancy records from your Django backend.

View API guide

Create

Send a POST request to add a new vacancy. Include JSON data for title, description, company, location, and salary.

POST /api/vacancies/ { "title": "Backend Developer", "company": "Amazing Hunting", "location": "Remote", "description": "Build and maintain Django APIs.", "salary": "5000" }

Read

Fetch open vacancies or details for a specific vacancy using GET requests. Use the list route to display all active jobs.

GET /api/vacancies/ GET /api/vacancies/{id}/

Update

Update an existing vacancy by sending a PUT or PATCH request to the vacancy detail endpoint with the fields you want to change.

PUT /api/vacancies/{id}/ { "title": "Senior Backend Developer", "salary": "6000" }

Delete

Remove a vacancy from the system with a DELETE request. This is ideal for closing positions that are no longer available.

DELETE /api/vacancies/{id}/

API usage examples

List vacancies
Use GET /api/vacancies/ to fetch all vacancies.

Create vacancy
Use POST /api/vacancies/ with JSON body.

Detail vacancy
Use GET /api/vacancies/{id}/ to get one vacancy.

Update vacancy
Use PUT /api/vacancies/{id}/ or PATCH /api/vacancies/{id}/ to change existing data.

Delete vacancy
Use DELETE /api/vacancies/{id}/ to remove the vacancy.
